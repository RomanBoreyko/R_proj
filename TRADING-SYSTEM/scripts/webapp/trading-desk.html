Юридический базис арбитражей — почему схождение ГАРАНТИРОВАНО
Арбитраж ≠ статистика. Схождение паритета/календаря/базиса обеспечено не «обычно так бывает», а спецификацией контракта: дата поставки, валюта расчёта, индекс. Здесь — выписка из живых спеков (через API) + ссылки на правила Bybit. Это верхняя строка карточки скрининга: КОНТРАКТ.

Опцион европейский = исполняется ТОЛЬКО в дату экспирации (нет раннего ассайнмента). Расчёт кэшем по индексу. Источник — спека контракта (кнопка ниже тянет live deliveryTime).

Закон паритета (Stoll 1969)

формулаC − P = F − K
синт-форвардK + C − P

Колл и пут ОДНОГО страйка и ОДНОЙ экспирации → синтетический форвард к этой дате. Если синт ≠ фьюч той же даты → разрыв, закрываемый контрактно на сеттле. Юр-база: оба гаснут в один deliveryTime к одному индексу в одной settleCoin.

⚖️ Юридический якорь каждого арбитража

АрбитражЧто гарантирует схождение (контрактно)
Паритет (синт↔фьюч)один deliveryTime + один индекс + одна settleCoin → в дату оба = индекс
Календарь (near↔far)каждый фьюч кэш-сеттлится к индексу в свой deliveryTime → базис→0 на сеттле
Cash-and-carry (фьюч↔перп/спот)фьюч сходится к индексу на сеттле; перп привязан фандингом к тому же индексу
Box (два паритета)выплата = (K2−K1) детерминирована на сеттле, независимо от цены БА

Везде якорь один: принудительный кэш-расчёт к индексу в зафиксированный deliveryTime. Это превращает «ставку на схождение» в «ожидание исполнения контракта».

🎯 Базовый актив = ИНДЕКС (общий знаменатель всех трёх)

ИнструментБазовый активМеханизм привязки
ОпционindexPriceкэш-сеттл к 30-мин среднему индекса @ deliveryTime
Дат. фьючindexPriceкэш-сеттл к индексу @ deliveryTime
ПерпindexPriceфандинг тянет lastPrice к индексу непрерывно

indexPrice ≠ Bybit-спот. Это композит спот-цен ETH/USDT по нескольким крупным биржам. markPrice (сглажен к индексу) — по нему считают PnL и ликвидацию, НЕ по lastPrice стакана. Все три обёртки сходятся не друг к другу, а к ОДНОМУ индексу — вот почему базис закрывается.

График: цена опциона / спред / vs БШМ / индекс — копится вживую (истории у Bybit нет)
Bybit API не отдаёт свечи опционов — нет endpoint kline для category=option. Поэтому ряд копится с момента «Старт»: опрос тикера(ов) раз в N секунд, точки в localStorage (переживают перезагрузку).
Одна нога (B пусто) → переключатель Вид: маркет (mark+bid/ask+греки) · vs БШМ (mark против справедливой цены Блэк-Шоулза при фикс. Реф-IV → разрыв = переоценка волой/вега) · индекс (indexPrice — общий якорь, к нему всё гаснет на экспирации).
Две ноги → спред A/B (как ETHUSDT.P/ETHUSDT в TV, но для опционов) с μ/σ/Z. Любую ногу — опцион / перп / фьюч / спот / инверс, миксуй. Пример: A=ETH-25DEC26-1700-P-USDT, B=ETH-25DEC26-1700-C-USDT, режим A/B×100. Опцион: …-P-USDT (пут) / …-C-USDT (колл); USDC-опционы — без суффикса.

График цены опциона

Одна нога · Вид: маркет (бирюза=mark, пунктир=bid/ask) · vs БШМ (бирюза=mark, янтарь=БШМ@refIV) · индекс (белая=indexPrice). Две ноги: фиолетовая=спред A/B, серый пунктир=μ±σ, бордовый=±2σ. Сбор идёт и на других вкладках.
