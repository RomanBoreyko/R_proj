# Trading Desk — документация (D1 арб / D2 ребаланс)

Локальное веб-приложение (один файл `trading-desk.html`, без сборки и зависимостей) для двух направлений: **D1** арбитраж под прикрытием опционов и **D2** гамма-скальп перп+пут, плюс чтение портфеля на Bybit.

Связано: [[03-STRATEGIES/D1-арбитраж-под-прикрытием-опционов]] · [[03-STRATEGIES/D2-eth-perp-put-ребаланс]] · [[09-INFRASTRUCTURE]]

## Запуск

CORS у Bybit работает только по **http**, не по `file://` — нужен локальный сервер. Только стандартный Python, ничего ставить не нужно.

**1. Сервер** (одна строка, `cd` не нужен — папка задана флагом `-d`):
```powershell
python -m http.server 8777 -d "D:\Vault\Project\R_projekt\R-trade\R_trade_proj_sin-arb\TRADING-SYSTEM\scripts\webapp"
```
**2. Открыть в браузере:** `http://localhost:8777/trading-desk.html`

**3. Остановить:** `Ctrl+C` в терминале.

**В одну вставку** (сервер + сам откроет браузер), PowerShell:
```powershell
Start-Process "http://localhost:8777/trading-desk.html"; python -m http.server 8777 -d "D:\Vault\Project\R_projekt\R-trade\R_trade_proj_sin-arb\TRADING-SYSTEM\scripts\webapp"
```
Браузер откроется сразу; если успел до старта сервера — обнови `F5`.

Заметки:
- Порт 8777 занят → поменяй на свободный (напр. `8778`) в **обеих** строках и в URL.
- `python` не найден → попробуй `py` вместо `python`.
- В `.claude/launch.json` конфиг для превью (легаси-имя `gold-calc`).

## Вкладки

### 1. Живые данные
Тикеры золотых инструментов (перпы XAUUSDT/XAUTUSDT/XAUTPERP, спот XAUTUSDT, датированные фьючи XAUTUSDT-*), базис спот↔перп, синт-спред XAUT−XAU, фандинг. Авто-обновление каждые 15с.

### 2. Синт-спред / стат-арб
Спред любой пары ног (перп↔перп, перп↔спот, фьюч↔спот):
- среднее, sigma, **Z-score**;
- уровни стренгла (центр ±kσ, крылья);
- **вола спреда** + перцентиль (дёшево/дорого продавать);
- **стат-арб**: полужизнь возврата к среднему (AR(1)), корреляция ног;
- **график спреда** с бэндами и золотой линией «собрано @» (уровень входа из позиций, по ключу).

### 3. Вол-арб (IV/RV)
RV из klines (live) против IV (ручной ввод) → IV/RV, сигнал продавать/покупать волу.

### 4. Календари
- **Фьючерсный** (live): датированные фьючи XAUTUSDT — спред far−near, roll yield годовых.
- **Опционный** (ручной ввод): near/far IV, тета, вега, премия → нетто дебет, тета, вега, APR.

### 5. Структура
Опционная конструкция (reverse iron condor / iron condor) — payoff + карта гаммы с зонами хеджа:
- Ввод: спот, шаг страйка, внутр./внешн. страйки, IV, дни, лотов, **тип** (long центр / short центр).
- **Payoff** на экспирацию с линиями страйков.
- **Карта гаммы** (Блэк-Шоулз, r=0): нетто-гамма по цене, зоны подсвечены — зелёные (+гамма → реверсия), красные (−гамма → продолжение), серые (кэп за крыльями).
- **Точки переключения** = реальные нуль-пересечения гаммы (не страйки!), с дедбендом.
- Таблица «карта хеджа по зонам» (data-driven) + подпись о зависимости от срока (ближе к экспирации гамма концентрируется у страйков).

### 6. Сайзинг / риск
Дельта-нейтральные размеры ног под целевой ноционал, маржа (спот = полный ноционал, перп = по марже).

### 7. Портфель 🔒
Чтение аккаунта по **read-only** API-ключу (ключ хранится только в localStorage браузера, подпись HMAC-SHA256 локально). Тянет все linear USDT/USDC позиции (вкл. серебро XAGUSDT), опционы с греками и кошелёк. Операционный план книги — [[03-STRATEGIES/portfolio-operating-plan]].
- **Мета-спред** — вся книга как одна позиция: нетто Δ$/Γ/Θ/Vega, поза, концентрация риска, интерактивный сценарий «рынок % + Δ IV → P&L».
- **Капитал** — эквити, IM/MM, запас по марже.
- **Нетто-греки** + греки по активам (с Δ$).
- **Спот / кошелёк** — $-оценка монет, % спота, флаг золотого спота.
- **Спреды / структуры** — группировка позиций по активу с распознаванием конструкций (стрэнгл/спред/календарь) и нетто-греками.
- **Состояние спредов** — bar-chart uPnL по структурам.
- **Спреды портфеля — графики** — авто-графики спреда по каждой паре из позиций.
- **График баланса USDT** — кривая из истории сделок (transaction-log).

### 8. График 📈
Цена опциона + **спред двух инструментов** + **vs БШМ** + **индекс** — всё **копится вживую**, потому что Bybit API **не отдаёт свечи опционов** (нет `kline` для `category=option`):
- **Две ноги A и B**, у каждой тип: опцион / перп-фьюч (linear) / спот / инверс — можно **миксовать** (опцион÷перп, опцион÷спот, фьюч÷фьюч и т.д.).
- **Режим**: `один (A)` · `A/B` · `A/B×100` · `A−B` · `(A−B)/B %` — как `ETHUSDT.P/ETHUSDT` в TV, но для опционов. Пример: `(ETH-25DEC26-1700-P-USDT / …-1700-C-USDT)×100`.
- «▶ Старт» → опрос тикера(ов) раз в N сек, точки в **localStorage** (`opt_series_*`, переживают перезагрузку; сбор идёт и на других вкладках). «🗑 Очистить» при смене связки.
- **Одна нога** — переключатель **Вид**: `маркет` (mark + bid/ask + греки) · `vs БШМ` (mark vs справедливая цена Блэк-Шоулза при фикс. **Реф-IV** → разрыв = переоценка волой/вега) · `индекс` (indexPrice — общий якорь, к нему всё гаснет на экспирации).
- **Две ноги**: спред A/B (фиолетовая) + μ±σ (серый) + ±2σ (бордовый) + статистика μ/σ/**Z** и значения ног.
- Символ опциона: `…-P-USDT` (пут) / `…-C-USDT` (колл); USDC-опционы — без суффикса `-USDT`.
- Это **коллектор истории** опционов/спредов — заменяет отсутствующий kline-эндпоинт. Копи ряд во времени для анализа арбитража/RV (синт-форвард, скос, паритет).

### 9. Оборот 🎯
Фарм наград за оборот / ребейтов под опционным прикрытием (прикрытый интрадей, D2). Использует **read-only** ключ из «Портфель»:
- «🔄 Подтянуть» → твой **реальный фи-тир** (signed `/v5/account/fee-rate`, перп+опцион — снимает вопрос 0.02 vs 0.04; мейкер < 0 = ребейт) + **оборот и фи сегодня** (`/v5/execution/list`, Σ execValue / Σ execFee, снимок последних ~100 филлов UTC).
- **Опц-кавер (стрэнгл)**: актив · DTE · крылья ±% · IV · размер → live-индекс + `bs()` считают премию, **тету/день** (автозаполняет «тета прикрытия» в EV) и гамму; сравнение со стрэддлом ATM (стрэнгл дешевле). Дешёвое D1-прикрытие.
- **EV-калькулятор**: целевой оборот · мейкер %/филл (автозаполн.) · награда $/М · тета прикрытия → `НЕТТО = награда − фи − тета` + порог награды на безубыток. Заработок — на наградах+ребейтах, не на скальпе.

## Безопасность ключей
- Только **read-only** права (Account/Position/Wallet/Order — чтение). Без Trade и Withdraw.
- Желательно привязка IP.
- Ключ не уходит никуда вовне — подпись считается в браузере, запросы идут напрямую на api.bybit.com. Кнопка «Забыть ключ» стирает из localStorage.

## Технически
- Чистый vanilla JS, canvas-графики (функции `lineChart`, `barChart`), без внешних библиотек.
- Bybit v5 API: `/v5/market/{tickers,kline,instruments-info}` (публичные), `/v5/account/{wallet-balance,transaction-log}`, `/v5/position/list` (подписанные).
- Опционов на золото на Bybit нет в market-data фильтре, но позиции по XAUT-опционам читаются через position/list (category=option).

> ⚠️ Все уровни — ориентиры, не приказ к исполнению.
