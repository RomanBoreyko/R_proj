# Gold Vol-Arb / Stat-Arb Calculator — документация

Локальное веб-приложение (один файл `gold-volarb-calculator.html`, без сборки и зависимостей) для анализа синтетического спреда золота, вол-арбитража, стат-арбитража, опционных/фьючерсных календарей и состояния портфеля на Bybit.

Связано: [[03-STRATEGIES/xau-synthetic-spread]] · [[09-INFRASTRUCTURE]]

---

## Запуск

CORS у Bybit работает только по **http**, не по `file://`. Поднять локальный сервер и открыть в браузере:

```powershell
python -m http.server 8777 -d "D:\Vault\Project\R_projekt\R-trade\R_trade_proj_sin-arb\TRADING-SYSTEM\scripts\webapp"
```
→ `http://localhost:8777/gold-volarb-calculator.html`

Только стандартный Python — ничего ставить не нужно. (В `.claude/launch.json` есть конфиг `gold-calc`.)

## Вкладки

### 1. Живые данные
Тикеры золотых инструментов (перпы XAUUSDT/XAUTUSDT/XAUTPERP, спот XAUTUSDT, датированные фьючи XAUTUSDT-*), базис спот↔перп, синт-спред XAUT−XAU, фандинг. Авто-обновление каждые 15с.

### 2. Синт-спред / стат-арб
Спред любой пары ног (перп↔перп, перп↔спот, фьюч↔спот):
- среднее, sigma, **Z-score**;
- уровни стренгла (центр ±kσ, крылья);
- **вола спреда** + перцентиль (дёшево/дорого продавать);
- **стат-арб**: полужизнь возврата к среднему (AR(1)), корреляция ног;
- **график спреда** с бэндами и золотой линией «собрано @» (уровень входа из позиций, по ключу).

### 3. Вол-арб (IV/RV)
RV из klines (live) против IV (ручной ввод) → IV/RV, сигнал продавать/покупать волу.

### 4. Календари
- **Фьючерсный** (live): датированные фьючи XAUTUSDT — спред far−near, roll yield годовых.
- **Опционный** (ручной ввод): near/far IV, тета, вега, премия → нетто дебет, тета, вега, APR.

### 5. Структура
Опционная конструкция (reverse iron condor / iron condor) — payoff + карта гаммы с зонами хеджа:
- Ввод: спот, шаг страйка, внутр./внешн. страйки, IV, дни, лотов, **тип** (long центр / short центр).
- **Payoff** на экспирацию с линиями страйков.
- **Карта гаммы** (Блэк-Шоулз, r=0): нетто-гамма по цене, зоны подсвечены — зелёные (+гамма → реверсия), красные (−гамма → продолжение), серые (кэп за крыльями).
- **Точки переключения** = реальные нуль-пересечения гаммы (не страйки!), с дедбендом.
- Таблица «карта хеджа по зонам» (data-driven) + подпись о зависимости от срока (ближе к экспирации гамма концентрируется у страйков).

### 6. Сайзинг / риск
Дельта-нейтральные размеры ног под целевой ноционал, маржа (спот = полный ноционал, перп = по марже).

### 7. Портфель 🔒
Чтение аккаунта по **read-only** API-ключу (ключ хранится только в localStorage браузера, подпись HMAC-SHA256 локально).
- **Мета-спред** — вся книга как одна позиция: нетто Δ$/Γ/Θ/Vega, поза, концентрация риска, интерактивный сценарий «рынок % + Δ IV → P&L».
- **Капитал** — эквити, IM/MM, запас по марже.
- **Нетто-греки** + греки по активам (с Δ$).
- **Спот / кошелёк** — $-оценка монет, % спота, флаг золотого спота.
- **Спреды / структуры** — группировка позиций по активу с распознаванием конструкций (стрэнгл/спред/календарь) и нетто-греками.
- **Состояние спредов** — bar-chart uPnL по структурам.
- **Спреды портфеля — графики** — авто-графики спреда по каждой паре из позиций.
- **График баланса USDT** — кривая из истории сделок (transaction-log).

## Безопасность ключей
- Только **read-only** права (Account/Position/Wallet/Order — чтение). Без Trade и Withdraw.
- Желательно привязка IP.
- Ключ не уходит никуда вовне — подпись считается в браузере, запросы идут напрямую на api.bybit.com. Кнопка «Забыть ключ» стирает из localStorage.

---

## Технически
- Чистый vanilla JS, canvas-графики (функции `lineChart`, `barChart`), без внешних библиотек.
- Bybit v5 API: `/v5/market/{tickers,kline,instruments-info}` (публичные), `/v5/account/{wallet-balance,transaction-log}`, `/v5/position/list` (подписанные).
- Опционов на золото на Bybit нет в market-data фильтре, но позиции по XAUT-опционам читаются через position/list (category=option).

> ⚠️ Все уровни — ориентиры, не приказ к исполнению.
