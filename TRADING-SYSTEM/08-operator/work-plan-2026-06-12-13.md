## ТЗ вкладки «базис-десک» (уточнено 11.06, ночная сессия)

Рабочая конструкция (4 ноги, нейтральная):
```
XAUT perp long q₁  +  Synth long q₂ (sell P K₁ + buy C K₂)  −  XAU perp q₃
q₃ = q₁ + Δsynth·q₂  →  нетто-Δ ≈ 0
Базисы: B₁ = XAUT−XAU (кросс)   B₂ = Synth−Perp (паритет, схождение контрактное)
```

### Формулы (блок в вкладке)
- Synth same-strike: Synth = K + C − P;  сплит-страйк: вход ∈ [K₁−D, K₂−D], D = P_bid(K₁)−C_ask(K₂)
- Исполняемые ширины: B_conv = Perp_bid − (K + C_bid − P_ask);  B_rev = (K + C_ask − P_bid) − Perp_ask
- Нейтрализатор: q₃ = q₁ + (Δc−Δp)·q₂
- Косты: 4 ноги ≈ 0.02%×2(опц) + 0.02−0.055%(перп×2) → полоса ±$
- BSM-теоретика от mid-IV → бэнды «выше/ниже БШМ» для поочерёдного котирования

### Калькуляторы
1. Нейтрализатор: ввод ног/количеств → нетто Δ$/Γ/V/Θ + рекомендуемый q₃ XAU
2. Лесенка: шаг по B (≥ костов), уровней, размер/уровень → маржа, макс. позиция,
   ожидание на схождении; добор БОЛЬШИМ объёмом при |B| > 2 шагов
3. Эдж-монитор: live B_conv/B_rev минус косты → зелёный/красный
4. БШМ-котировщик: страйк → theo vs bid/ask, подсветка bid>theo (продать) / ask<theo (купить)

### Графики
- TV-виджет (живой): BYBIT:XAUTUSDTPERP/BYBIT:XAUUSDTPERP — только кросс-базис B₁
  (опционных ног в TV НЕТ и не будет — не пытаться)
- Canvas (самопальный): B₂ тиками из 3 стаканов, 2 линии + полоса костов,
  история localStorage
- Excel: лист PARITY в учётной ексели (День 2) — снапшоты time/F/K/C/P/B_conv/B_rev,
  дописываются скриптом parity_log.py → родные график Excel по логу

### Заметки
- RFQ Builder: мин. 10 XAUT (~$40k) — инструмент будущего масштаба, пока стаканы лимитками
- Bybit payoff-чарт в RFQ ≠ график базиса; Day-0 P&L по маркам — иллюзия до котировки LP
- Эскалация: скальп лесенкой → календарь синтетиков → стрэнгл-обёртка, ребаланс перпами
