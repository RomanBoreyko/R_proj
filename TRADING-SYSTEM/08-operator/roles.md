# РОЛИ — функции на сервере мясного скафандра

> Один человек, семь функций. В каждый момент времени активна ОДНА.
> Смысл: на время становиться функцией, а не человеком с роем суеты в голове.
> Конечная цель: фемили-офис (мать + её сестра), роли делегируются живым людям.

## Протокол смены роли

```
1. Закрыть артефакты текущей роли (вкладки, файлы, стаканы)
2. Вслух/письменно: «Выхожу из <роли>. Вхожу в <роль>.»
3. Открыть ТОЛЬКО артефакты новой роли
4. Запрещено: решения чужой роли. Хочется — записал в инбокс той роли, не сделал.
```

Тег роли ставится в коммиты и заметки. Узлы ролей (инбоксы там):

## 1. 🎛 ОПЕРАТОР ПОРТФЕЛЯ — [OP]

**Функция:** исполнение по системе. Входы, выходы, ребалансы, ролл.
**Артефакты:** Bybit терминал, вкладка «Портфель», risk-rules.
**Решения:** только внутри правил [[06-RISK/risk-rules]]. Лесенки, лимитки, крылья.
**Запрещено:** менять правила на лету; «не удержался, захотелось» (кейс VELVET);
торговать во время работы других ролей; маркет-ордера без причины.
**KPI:** 0 нарушений правил за сессию; все ноги с покрытием; IM < 50%.

## 2. 🔧 РАЗРАБОТЧИК ИНФРАСТРУКТУРЫ — [DEV]

**Функция:** строит инструменты Оператора: калькулятор, скрипты, Pine, панель паритета.
**Артефакты:** VSCode/Claude Code, webapp, scripts/, parity_scan, analyze_book.
**Решения:** архитектура инструментов, формулы, UX вкладок.
**Запрещено:** открывать позиции («просто проверить идею» — записать Оператору в инбокс);
рефакторить во время торговой сессии Оператора.
**KPI:** инструмент закрывает задачу из бэклога; ноль сломанных вкладок.

## 3. ⚙️ ДЕВОПС — [OPS]

**Функция:** развёртывание и живучесть: сервер калькулятора, git, бэкапы,
секреты, MetaScalp-привод, порты, мониторинг.
**Артефакты:** http.server 8777, git/GitHub, .bybit/credentials, metascalp_probe,
.gitmodules, .gitignore.
**Решения:** что в git/что в ignore, когда коммитить, структура репо.
**Запрещено:** git add -A; секреты в репо; снос чужих проектов (сабмодули!).
**KPI:** калькулятор поднимается одной командой; репо чистый; ключи вне git.

## 4. 📋 ПРОЕКТНЫЙ МЕНЕДЖЕР — [PM]

**Функция:** документация = единая точка правды. Планы, INDEX, линковка графа,
чистка мусора, согласованность нот между собой.
**Артефакты:** Obsidian, TRADING-SYSTEM/*.md, work-plan, списки вопросов.
**Решения:** структура доков, что слить/удалить, версии нот.
**Запрещено:** писать код; торговать; превращать ноты в музей (если нота мертва — слить или удалить).
**KPI:** ноль сирот в графе; INDEX = реальность; план дня закрыт чекбоксами.

## 5. 🧾 КОНСИЛЬЕРИ — [CON]

**Функция:** память и совесть портфеля. Отчёты по транзакциям Оператора,
таблица позиций, утренний/вечерний срез, отпечатывание состояния книги в голове.
**Артефакты:** TRADING_BOOK / учётная ексель (ручной журнал!), analyze_book,
recent_ops, pnl_breakdown, лист PARITY.
**Решения:** формат учёта, что считать аномалией, когда бить тревогу Оператору.
**Запрещено:** торговать; оправдывать сделки Оператора задним числом — только факты.
**KPI:** каждая операция записана в течение дня; расхождение журнал↔биржа = 0;
утренний срез до первой сделки.
**Ритуал:** вечером вслух пересказать книгу по памяти, потом сверить с таблицей.

## 6. 🧠 ОПЕРАТОР НЕЙРОНОК — [NEO]

**Функция:** интерфейс к Claude/моделям. Формулировка задач, контекст, проверка
выводов, ведение памяти проекта, распределение задач по ролям через Claude.
**Артефакты:** Claude Code, Claudesidian, memory, промпты, списки вопросов.
**Решения:** что делегировать модели / что делать руками; качество ТЗ.
**Запрещено:** слепо исполнять выводы модели в рынке (модель — советник Консильери,
не Оператор); скармливать секреты в чат.
**KPI:** задача сформулирована так, что модель решает с первого захода;
память актуальна.

## 7. 📈 СПРЕДЕР — [SPR]

**Функция:** визуализация спредов. Графики в TV / MT5 / MetaScalp / Bybit /
Excel (если всё совсем плохо). Разметка, алерты, ratio-формулы, Pine.
**Артефакты:** TradingView (PERP-символы, ratio), MT5, MetaScalp-кластеры,
canvas-графики калькулятора, Excel-чарты PARITY.
**Решения:** какой спред на каком слое смотреть (TV=кросс-базис, canvas=паритет,
Excel=постфактум-разбор); уровни алертов.
**Запрещено:** торговать с графика мимо Оператора; плодить 40 графиков без алертов.
**KPI:** каждый активный спред имеет график + алерт; раскорреляция замечена
ДО того, как Оператор спросил.

## Зачем это работает (механика, не мистика)

Роль = ограничение пространства решений. Меньше решений → меньше шума → глубже фокус.
Нарушения правил происходят на СТЫКАХ (Оператор ночью становится дегенератом —
кейс VELVET −$20). Явная смена роли с ритуалом = файрвол между функциями.
В фондах эти роли разделены людьми не от богатства, а потому что это работает.
Фемили-офис начинается не с денег — с того, что роли уже описаны и работают.
