# ФОРМУЛЫ — ПРИБЫЛЬ И РИСК

## I. СЕТОЧНЫЙ БОТ

### Прибыль одной ячейки (Arithmetic)

```
Шаг_$ = (Upper - Lower) / N

Прибыль_ячейки = Шаг_$ - Fee_вход - Fee_выход
               = Шаг_$ - 2 × Taker_rate × Цена_входа × Размер

где:
  N           = количество ячеек
  Taker_rate  = 0.00055 (Bybit USDT perp taker)
  Maker_rate  = 0.00002 (Bybit USDT perp maker)
```

### Прибыль одной ячейки (Geometric)

```
Ratio = (Upper / Lower)^(1/N)
Прибыль_% = (Ratio - 1) - 2 × Fee_rate
```

### APR сетки

```
Срабатываний_в_день = ATR_дневной / Шаг_$
APR = Прибыль_ячейки × Срабатываний_в_день × 365 / Инвестиция × 100%
```

### Минимальный шаг для безубыточности

```
Шаг_min = 2 × Taker_rate × Цена
(Шаг должен быть > Шаг_min)
```

## II. ОПЦИОННАЯ ОБЁРТКА

### Тета-доход (ежедневный)

```
Theta_day = ∂V/∂t  (из Greeks опциона)

Нетто_тета = Theta_продан_CALL + Theta_продан_PUT - Theta_куплен_PUT
Доход_тета_в_день = Нетто_тета × Размер_контракта
```

### Стоимость конструкции (нетто-дебет/кредит)

```
Нетто = Премия_CALL_продан + Премия_PUT_продан - Премия_PUT_куплен

Если Нетто > 0 → кредитная конструкция (получаем при входе)
Если Нетто < 0 → дебетовая (платим, но защита больше)
```

### APR опционной обёртки

```
APR_options = (Нетто_в_мес × 12) / Объём_бота × 100%
```

### Максимальный убыток конструкции

```
Сценарий пробоя вниз:
  Убыток_PUT_spread = (Strike_PUT_куплен - Strike_PUT_продан) × Лот - Нетто_кредит

Сценарий пробоя вверх:
  Убыток_CALL = (Цена - Strike_CALL) × Лот - Премия_CALL
  (НО: сетка лонг → прибыль сетки частично компенсирует)
```

## III. БАЗИСНЫЙ АРБИТРАЖ (Спот–Фьюч)

### Базис

```
Basis = Фьюч_цена - Спот_цена
Basis% = (Basis / Спот_цена) × 100%

Annualized_basis = Basis% × (365 / Дней_до_экспирации)
```

### Позиция: купить спот + продать фьюч

```
Entry: Basis > Threshold (напр. > 0.5% annualized)
Exit: Basis → 0 (при экспирации всегда 0)

PnL = Basis_вход - Basis_выход - Costs
Costs = Комиссия_спот + Комиссия_фьюч + Проскальзывание
```

> ⚠️ Шортов фьюча нет! → Базисный арб только через Deribit (там продаём фьюч против байбит-спота)

## IV. КАЛЕНДАРНЫЙ СПРЕД

### P&L компоненты

```
Theta_PnL = (Theta_ближний - Theta_дальний) × Δt
Vega_PnL  = (Vega_дальний - Vega_ближний) × ΔIV
Gamma_PnL = (Gamma_дальний - Gamma_ближний) × ΔS² / 2

Итого_PnL = Theta_PnL + Vega_PnL + Gamma_PnL
```

### Break-even движение

```
BE_движение = √(2 × |Theta_net| / Gamma_net)
(движение цены, при котором гамма-убыток = тета-доход)
```

## V. РИСК-ФОРМУЛЫ

### Маржа и ликвидация

```
Начальная маржа = Объём × Цена × Ставка_маржи
Ставка_маржи = 1 / Плечо

Цена_ликвидации (LONG) = Цена_входа × (1 - 1/Плечо + Ставка_обслуживания)
                        = Цена_входа × (1 - (Плечо-1)/Плечо)

При плече 2×: Цена_ликвид = Цена_входа × 0.5 (упрощённо)
```

### Правило: нет ликвидации

```
Нижняя_сетки > Цена_ликвидации × 1.3 (30% буфер)
→ При плече 2×: Нижняя_сетки > Цена_входа × 0.65
```

### Максимальный убыток системы (всего портфеля)

```
Max_Loss = Σ(по всем ботам) Max_убыток_бота_при_пробое_вниз
         = Σ [Инвестиция_бота × (1 - Lower/Upper_entry)]
           - Σ Прибыль_опционных_защит
```

### Позиционный лимит

```
Размер_бота ≤ Портфель × 0.15 (не более 15% на один бот)
Суммарное_плечо ≤ 1.5× (среднее по портфелю)
```

## VI. РЕБАЛАНСИРОВКА

### Триггер

```
Отклонение% = |Текущий_вес_актива - Целевой_вес| / Целевой_вес
Ребаланс если Отклонение% > 5%
```

### Размер ребалансировки

```
Δ_USDT = (Целевой_вес - Текущий_вес) × Общий_портфель
Продаём переросший актив, покупаем отставший
```

### Прибыль от ребалансировки (теоретическая)

```
Rebalancing_bonus ≈ σ² × (1 - ρ) / 2  (в год, для двух активов)
где:
  σ = средняя волатильность активов
  ρ = корреляция между активами
```

## VII. АРБИТРАЖ IV

```
IV_спред = IV_инструмент_A - IV_инструмент_B
Vega_PnL = IV_спред × Vega × Лот

Порог входа: IV_спред > 3%
Порог выхода: IV_спред < 1%
Break_even: IV_спред > 2 × Транзакционные_издержки_в_vol_терминах
```
