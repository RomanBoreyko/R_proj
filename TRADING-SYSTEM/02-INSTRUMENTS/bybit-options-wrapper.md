# BYBIT — ОПЦИОННАЯ ОБЁРТКА СЕТОЧНЫХ БОТОВ

## Концепция

Каждый дневной и недельный бот окружён опционной конструкцией.
Опции привязаны к границам сетки.

```
         ┌──────────────────────────────────────┐
         │  Куплен PUT (защита низа)             │
         │  Strike = Нижняя_сетки × 0.95        │
         ├──────────────────────────────────────┤
         │                                      │
         │     СЕТОЧНЫЙ БОТ                     │
         │     [Lower_Price ... Upper_Price]     │
         │                                      │
         ├──────────────────────────────────────┤
         │  Продан CALL (доход выше диапазона)  │
         │  Strike = Верхняя_сетки × 1.05       │
         │                                      │
         │  Продан PUT (доход ниже диапазона)   │
         │  Strike = Нижняя_сетки × 0.90        │
         └──────────────────────────────────────┘
```

---

## Три ноги конструкции

### Нога 1: Купленный PUT (защита)
- **Роль**: страховка от пробоя сетки вниз
- **Strike**: `Нижняя_сетки × 0.95` (на 5% ниже нижней)
- **Экспирация**: конец месяца / конец года
- **Размер**: соответствует объёму бота (1:1 по дельте при пробое)
- **Стоимость**: платим премию — это страховка

```
Защита_PUT:
  Если цена < Strike_PUT:
    Прибыль_пута ≈ (Strike_PUT - Цена_текущая) × Лот
```

### Нога 2: Проданный CALL (доход выше)
- **Роль**: доход если цена не вырастет выше верхней границы
- **Strike**: `Верхняя_сетки × 1.05` (на 5% выше верхней)
- **Экспирация**: конец месяца / конец года
- **Размер**: 50–75% от объёма бота (частичный хедж дельты)
- **Риск**: если цена пробьёт вверх → убыток от кола, но сетка в прибыли

```
Доход_CALL_продан = Премия_кола × Лот (если цена < Strike_CALL до экспирации)
```

### Нога 3: Проданный PUT (доход у нижней)
- **Роль**: дополнительная премия + снижение цены входа
- **Strike**: `Нижняя_сетки × 0.90` (на 10% ниже нижней)
- **Экспирация**: конец месяца
- **Риск**: обязан купить если цена упадёт ниже → только если есть USDT резерв

```
Доход_PUT_продан = Премия_пута × Лот (если цена > Strike_PUT_продан до экспир)
Риск_PUT_продан = (Strike_PUT_продан - Цена) × Лот (если пробой)
```

## Принципы защиты

```
Купленный PUT strike > Проданный PUT strike (всегда)
  → Put Spread: убыток ограничен

Макс_убыток_put_spread = (Strike_PUT_куплен - Strike_PUT_продан) × Лот - Нетто_премия
```

---

## Доходность опционной обёртки

```
Нетто_премия_в_мес = Премия_CALL_продан + Премия_PUT_продан - Премия_PUT_куплен

При диапазоне не пробитом:
  Доход_обёртки = Нетто_премия_в_мес

APR_обёртки = (Нетто_премия × 12) / Объём_бота × 100%
```

## Временны́е горизонты обёртки

| Таймфрейм бота | Опционная обёртка |
|----------------|-------------------|
| 1H бот | Обёртки нет (слишком дорого) |
| 1D бот | Опционы с экспирацией конец месяца |
| 1W бот | Опционы с экспирацией конец года |

## Bybit Options API

```
Рынок опционов:
  GET /v5/market/tickers?category=option    → текущие IV и цены
  GET /v5/market/instruments-info?category=option → доступные страйки

Торговля:
  POST /v5/order/create (category=option)   → создать опционный ордер

Параметры ордера:
  category: "option"
  symbol: "ETH-28MAR25-2800-C"   (актив-экспир-страйк-тип)
  side: "Buy" / "Sell"
  orderType: "Limit"
  qty: кол-во контрактов
  price: цена в USDT
```

Документация опционов Bybit: https://bybit-exchange.github.io/docs/v5/option/option-spec
