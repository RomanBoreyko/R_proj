# DERIBIT — СТРАТЕГИИ

## Роль Deribit в системе
- Арбитраж волатильности vs Bybit
- Календарные спреды (торговля временем через term structure)
- Гамма-скальпинг
- Раскорреляции IV между активами

---

## 1. Календарный спред

### Концепция
```
Продать ближнюю экспирацию (высокая IV, быстрый тета-распад)
Купить дальнюю экспирацию (низкая IV, медленный распад)

Прибыль = разница в тета-распаде двух опционов
```

### Параметры входа
```
Вход: когда term_structure нормальная (IV_ближний > IV_дальний)
Выход: при экспирации ближнего опциона ИЛИ при инверсии структуры

Profit_max при цене = Strike на экспирацию ближнего
```

### Формула P&L

```
PnL_Кал_спред = Theta_ближний × Δt - Theta_дальний × Δt
              ± Vega_нетто × ΔIV
              ± Gamma_нетто × ΔS²/2
```

### Deltaнейтральность
```
Купить/продать фьюч для нейтрализации нетто-дельты:
  Хедж = -(Delta_продан + Delta_куплен)
  Пересчёт каждые 24H или при ΔS > 2%
```

## 2. Арбитраж IV: Deribit vs Bybit

### Когда работает
```
IV_Deribit(ETH, страйк X, экспир T) ≠ IV_Bybit(ETH, страйк X, экспир T)
Спред IV > 3% → торгуем
```

### Конструкция
```
Если IV_Deribit > IV_Bybit:
  Продать опцион на Deribit
  Купить опцион на Bybit
  Нейтрализовать дельту фьючом

Прибыль = (IV_Deribit - IV_Bybit) × Vega × Лот - транзакционные издержки
```

### Риски
- Маржинальные требования на двух биржах
- Разница в ценах фьючей (базис)
- Разница ликвидности (проскальзывание)

## 3. Гамма-ребалансировка

### Концепция
```
Купить стрэддл (ATM call + put) = лонг гаммы
Хеджировать дельту фьючом при каждом движении актива

Прибыль = Gamma × ΔS² / 2 - Theta × Δt
Входить когда: Realized_Vol > Implied_Vol (vol дешева)
```

### Точки хеджирования
```
Метод 1: По времени — каждые N часов
Метод 2: По движению — каждые K% изменения цены
Метод 3: По гамма-бюджету — пересчёт при изменении дельты на X

Прибыль_хеджа = Γ × (ΔS)² / 2
```

## 4. Раскорреляции по активам

### XAUT vs BTC/ETH vol
```
XAUT (золото) обычно имеет низкую корреляцию с крипто
При стресс-событиях корреляция растёт → арбитраж раскорреляции

Стратегия:
  Продать vol на XAUT (если IV завышена относительно RV)
  Купить vol на ETH (если IV занижена относительно RV)
```

### Метрика раскорреляции
```
Ratio = IV_XAUT / IV_ETH (историческое среднее = X)
Вход: Ratio > X × 1.15 → продать XAUT vol, купить ETH vol
Выход: Ratio → X
```

## Deribit API

```
Базовый URL: https://www.deribit.com/api/v2/

Ключевые методы:
  public/get_book_summary_by_currency → все опционы по активу + IV
  public/get_index_price             → спот-цена
  public/ticker                      → bid/ask/IV/Greeks
  private/buy / private/sell         → ордера
  private/get_position               → позиции

Аутентификация: HMAC SHA256, API key + secret
WebSocket: wss://www.deribit.com/ws/api/v2/
```

Документация: https://docs.deribit.com/
