## Абсолютные запреты

```
❌ Шорты (фьюч или спот) — ЗАПРЕЩЕНО
❌ Плечо > 2× — ЗАПРЕЩЕНО
❌ Ликвидационный уровень в диапазоне сетки — ЗАПРЕЩЕНО
❌ Использование фандинга как основной стратегии — ЗАПРЕЩЕНО
❌ Более 15% портфеля в одном боте — ЗАПРЕЩЕНО
```

## Правило ликвидации

### Проверка перед запуском любого бота

```
Цена_ликвид_Long = Entry_price × (1 - InitMarginRate + TakerFeeRate)
где:
  InitMarginRate = 1 / Leverage
  TakerFeeRate = 0.00055

При плече 2×:
  Цена_ликвид ≈ Entry_price × 0.5

Нижняя_сетки должна быть: Нижняя_сетки > Цена_ликвид × 1.3
→ Нижняя_сетки > Entry_price × 0.65 (при плече 2×)
```

### Практическое правило

```
Плечо 1× → ликвидация невозможна (нет левериджа)
Плечо 1.5× → Нижняя ≥ Entry × 0.50
Плечо 2× → Нижняя ≥ Entry × 0.65

Рекомендация: XAUT = 1× плечо (золото, защитный актив)
              Остальные = 1.5× максимум в основном режиме
              2× только для краткосрочных 1H ботов с узкими сетками
```

## Правило диапазонов

```
Перед запуском:
1. Взять ATR(20) нужного таймфрейма
2. Lower = Текущая цена - 1.5 × ATR
3. Upper = Текущая цена + 1.5 × ATR
4. Проверить: Lower > Цена_ликвидации × 1.3
5. Если нет → уменьшить плечо или сузить диапазон

Пересмотр диапазонов:
  1H боты → каждые 3 дня
  1D боты → каждые 2 недели
  1W боты → раз в квартал
```

## Лимиты на платформах

| Платформа | Макс % портфеля | Макс плечо |
|-----------|----------------|------------|
| Bybit (сетки) | 60% | 2× |
| Bybit (споты) | 30% | 1× |
| Deribit | 20% | 1× (опции покрытые) |
| Coinbase | 20% | 1× |

> Суммарный портфель 100% = Bybit + Deribit + Coinbase
> Буфер ≥ 10% в стейблах на случай пополнения маржи

## Правило опционного резерва

```
Для каждого проданного PUT:
  Резерв_USDT = Strike_PUT × Лот × 1.1 (10% запас)

Этот резерв держать в Bybit Earn (депозит под %)
Не использовать этот резерв для других позиций
```

## Стресс-тест портфеля

### Сценарий: рынок -30% за неделю

```
Сетки:
  Lower_1D бот = -1.5×ATR_1D от текущей
  При падении 30% → большинство 1D ботов ниже Lower → остановка
  Убыток = Инвестиция × (Lower/Entry - 1) = ~15–20% от объёма бота

Опционная защита:
  PUT куплен на Strike = Lower × 0.95
  Прибыль PUT = max(0, Strike_PUT - Цена) × Лот
  → компенсирует часть убытка сетки

Итоговый макс убыток портфеля при -30% рынка:
  ~8–12% от всего портфеля (при правильных размерах)
```
