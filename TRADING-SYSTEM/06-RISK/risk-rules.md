## Абсолютные запреты

```

   другой ногой спреда, крылом, базой (конверсия) или

❌ Ликвидационный уровень в диапазоне сетки — ЗАПРЕЩЕНО
❌ Использование фандинга как основной стратегии — ЗАПРЕЩЕНО
❌ Более 15% портфеля в одном боте — ЗАПРЕЩЕНО
```

## Философия портфеля: мета мульти-спред

> Портфель — это **мета мульти-спред**: совокупность спредовых конструкций
> по многим активам, где убыток ограничен **статической репликацией**
> (крылья, противоноги, лимитники на краях вместо опционов),
> а прибыль динамическая — за счёт **динамического хеджа**
> (ребалансировка дельты, гамма-скальп, перезаход ног).

```
Ограниченный убыток   = статика: крыло / противонога / лимитник на краю
Динамическая прибыль  = динамика: ребаланс дельты, откуп премии, ролл
```

### Рабочие приёмы (что реально делается)

```
1. СКАЛЬП ПОД ПОДУШКУ ОПЦИОНА
   Когда опцион «освобождается» (его роль в конструкции закрыта,
   но он ещё жив) — против него скальпируем базой.
   Опцион = подушка скальпа (ограничивает риск перп-ноги).
   Цель: скальпом отбить премию опциона →
   опцион остаётся до экспирации бесплатным («а вдруг стрельнет»).
   Это ЕДИНСТВЕННЫЙ легальный способ досидеть купленный до экспирации
   вне конверсий и крыльев.

2. ПРОДАЖА КРАЁВ ПРОТИВ ФЬЮЧА
   Когда освобождается фьюч — против него продаём края
   (OTM-опционы): фьюч покрывает одну сторону проданного,
   получается covered-структура, премия капает.

3. СКАЛЬП ОПЦИОННЫХ СТАКАНОВ
   Неликвидные стаканы (XAUT, MNT, дальние серии) — лимитники
   внутри спреда, заработок на ширине. Только лимитками,
   только в плюс от BSM-теоретики.

4. СТАТИЧЕСКАЯ РЕПЛИКАЦИЯ ГДЕ НЕТ ОПЦИОНОВ
   На парах без опционного рынка (HUSDT.P/HUSDC и т.п.)
   края прикрываются лимитниками на продолжение движения —
   реплика опционного спреда: ограниченная прибыль/убыток,
   одна нога лонг, другая шорт.
```

### Статус: система оттачивается, инфраструктура готовится. Правила ниже — действующие, лимиты — целевые.

## Правило ликвидации

### Проверка перед запуском любого бота

```
Цена_ликвид_Long = Entry_price × (1 - InitMarginRate + TakerFeeRate)
где:
  InitMarginRate = 1 / Leverage
  TakerFeeRate = 0.00055

При плече 2×:
  Цена_ликвид ≈ Entry_price × 0.5

Нижняя_сетки должна быть: Нижняя_сетки > Цена_ликвид × 1.3
→ Нижняя_сетки > Entry_price × 0.65 (при плече 2×)
```

### Практическое правило

```
Плечо 1× → ликвидация невозможна (нет левериджа)
Плечо 1.5× → Нижняя ≥ Entry × 0.50
Плечо 2× → Нижняя ≥ Entry × 0.65

Рекомендация: XAUT = 1× плечо (золото, защитный актив)
              Остальные = 1.5× максимум в основном режиме
              2× только для краткосрочных 1H ботов с узкими сетками
```

## Правило диапазонов

```
Перед запуском:
1. Взять ATR(20) нужного таймфрейма
2. Lower = Текущая цена - 1.5 × ATR
3. Upper = Текущая цена + 1.5 × ATR
4. Проверить: Lower > Цена_ликвидации × 1.3
5. Если нет → уменьшить плечо или сузить диапазон

Пересмотр диапазонов:
  1H боты → каждые 3 дня
  1D боты → каждые 2 недели
  1W боты → раз в квартал
```

## Стресс-тест портфеля

### Сценарий: рынок -30% за неделю

```
Сетки:
  Lower_1D бот = -1.5×ATR_1D от текущей
  При падении 30% → большинство 1D ботов ниже Lower → остановка
  Убыток = Инвестиция × (Lower/Entry - 1) = ~15–20% от объёма бота

Опционная защита:
  PUT куплен на Strike = Lower × 0.95
  Прибыль PUT = max(0, Strike_PUT - Цена) × Лот
  → компенсирует часть убытка сетки

Итоговый макс убыток портфеля при -30% рынка:
  ~8–12% от всего портфеля (при правильных размерах)
```

## Правила опционной книги (из разбора статистики 2026-06-10)

> Источник: P&L Analysis Bybit за 03–09.06.2026. Активно закрытые сделки: **93% win, +$63.67** (продажа премии 21W/1L). Доведённые до экспирации: **28% win, −$44.04** (купленные лонги 5W/11L). По грекам за неделю: Gamma +$57, Delta +$39, **Theta −$45**, Vega ~0.

### Диагноз
```
Эдж:    продажа премии с активным откупом (гамма-скальп)
Утечка: купленные OTM-опционы, досиживающие до экспирации
        → сгорание съедает ~70% заработка от продажи
        → тета −$6.5/день платится 24/7, гамма зарабатывает
          только когда рынок ходит
```

### Правила

```
1. ВРЕМЕННОЙ СТОП НА КУПЛЕННЫЕ ОПЦИОНЫ
   Купленный опцион не в плюсе к половине своей жизни
   → продать остаток временной стоимости. До экспирации
   досиживают ТОЛЬКО ноги конверсий и крылья проданных структур.

2. ПОКУПКА ТОЛЬКО С РОЛЬЮ
   Каждый купленный опцион при входе получает роль:
   [крыло] покрытие проданной ноги — сгорание = плата за кэп, ОК
   [конверсия] лочится базой до экспирации, ОК
   [гамма] ставка на движение — обязателен временной стоп (п.1)
   Голые покупки «на движение» без сигнала (Z, сжатие волы) — НЕ открывать.
   Статистика тайминга покупок: 28% win — тайминг не работает.

3. ТЕТА-БАЛАНС КНИГИ
   Цель: Θ книги ≈ 0 (проданные дальние серии оплачивают
   купленные ближние). Допустимо: |Θ| < 1% эквити/день.
   Θ < −2% эквити/день → резать лонг-гамму или допродавать премию.

4. ЛОНГ-ГАММА ТОЛЬКО ПОД ДВИЖЕНИЕ
   В боковике лонг-гамма спит, тета капает.
   Покупать выпуклость ПЕРЕД движением (сигнал: |Z| > 2,
   сжатие RV, событие), не держать фоном.

5. НАПРАВЛЕННЫЙ ЛИМИТ
   Дельта-ставки работают (Delta +$39/нед) — но им явный лимит:
   |Δ$ книги| ≤ 20% эквити, преднамеренно, не «случайно»
   внутри опционных конструкций.

6. ГОЛЫЕ ПРОДАННЫЕ НОГИ — ВСЕГДА С КРЫЛОМ
   Проданный опцион без кэпа держит IM и несёт хвост
   (кейс BTC-25DEC26: IM 92%, голый 60000-C).
   Крыло покупается сразу при продаже, не «потом».
```
