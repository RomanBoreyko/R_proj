## СБОР СПРЕДОВ

### Типы спредов в системе

| Спред | Инструменты | Источник |
|-------|------------|---------|
| Basis спред | Спот vs Фьюч (Bybit/Deribit) | Контанго/бэквардация |
| IV спред | Bybit options vs Deribit options | Разница ценообразования |
| Calendar IV спред | Ближний vs дальний опцион | Term structure |
| Cross-asset vol спред | XAUT_IV vs ETH_IV | Раскорреляция активов |
| Grid спред | Buy/Sell внутри сетки | Bid-ask маркетмейкинг |

---

### Grid спред (маркетмейкинговая логика)

```
Сеточный бот = пассивный маркетмейкер

Buy limit @ Lower_cell
Sell limit @ Upper_cell

Спред_на_ячейку = Upper_cell - Lower_cell
Прибыль_после_fees = Спред_на_ячейку - 2 × Fee × Price

Система собирает этот спред каждый раз, когда цена проходит ячейку
```

### Basis спред (спот-фьюч)

```
Шаги:
1. Купить спот актива на Coinbase (или Bybit spot)
2. Продать фьюч на Deribit (не Bybit! — нет шортов на Bybit)
3. Держать до экспирации фьюча

Прибыль = Basis_вход × Позиция
         = (Фьюч_цена - Спот_цена)_при_входе × Лот

Риск: Basis может расшириться до экспирации (временный убыток)
      При экспирации Basis → 0 всегда
```

## ВОЛАТИЛЬНОСТЬ СПРЕДОВ

### Концепция
```
Implied Volatility (IV) = цена страха/неопределённости
Realized Volatility (RV) = фактическая историческая волатильность

IV спред = IV - RV

Если IV > RV (нормально): опционы переоценены → продаём
Если IV < RV (редко): опционы недооценены → покупаем
```

### Сбор vol-премии

```
Позиция: продать strangle (OTM call + OTM put)
  Прибыль ≈ (IV - RV) × Vega × Лот × √(T/252)

Рыночные данные:
  IV → с биржи (Deribit/Bybit)
  RV = √(252/N × Σ(ln(S_t/S_{t-1})²)  — считаем сами

Вход: IV/RV ratio > 1.2 (IV завышена на 20%)
Выход: Экспирация или IV/RV → 1.0
```

## ГАММА-РЕБАЛАНСИРОВКА

### Что такое гамма портфеля

```
Gamma (Γ) = скорость изменения дельты при движении цены

Гамма > 0 (лонг гамма): портфель выигрывает от движения (в любую сторону)
Гамма < 0 (шорт гамма): портфель проигрывает от движения, зарабатывает на покое

Наша система:
  Сетки = шорт гамма (зарабатывают в боковике)
  Купленные PUT = лонг гамма (зарабатывают при падении)
  Проданные опции = шорт гамма
  Нетто гамма ≈ небольшой шорт (получаем тету, платим при движении)
```

---

### Гамма-хеджирование фьючерсного спреда

```
Позиция: Купить опцион на Deribit (лонг гамма)
         Продать/купить фьюч для нейтрализации дельты

Каждые N часов или при ΔS > K%:
  Δ_новая = delta(опцион, новая_цена)
  Хедж = -(Δ_новая - Δ_старая) × Лот  → продать/купить фьюч

Прибыль_за_хедж = Γ × (ΔS)² / 2
Стоимость_хеджа = Theta × Δt

Гамма-торговля прибыльна когда: Реализованная_vol > Подразумеваемая_vol
```

### Гамма-ребалансировка спот-портфеля

```
Это НЕ опционная гамма — это аналогия

Портфель 5 активов в равных долях:
  При движении одного актива → доля меняется
  Ребалансировка = "гамма-скальпинг" спот-портфеля

При росте актива A на 10%:
  Его доля выросла → продаём часть A, покупаем другие
  → автоматически продаём дорогое, покупаем дешёвое

Прибыль_ребаланса ≈ σ²_портфеля × (1-ρ̄) / 2 × в_год
где ρ̄ = средняя попарная корреляция активов
```

---

### Частота ребалансировки

```
Чаще = больше транзакционных издержек
Реже = больший дрейф от целевых весов

Оптимум (практика):
  Триггер по пороговому отклонению: Δвес > 5%
  Минимальный интервал: 7 дней (не чаще)
  Максимальный интервал: 30 дней (не реже)
```
