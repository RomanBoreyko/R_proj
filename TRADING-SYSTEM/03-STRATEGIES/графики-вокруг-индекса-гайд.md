## Принцип

Все деривативы одного актива — обёртки вокруг ОДНОГО индекса (`indexPrice`,
композит спотов). Опцион/фьюч гаснут к индексу, перп привязан фандингом. Поэтому
осмысленный график показывает не цены, а **отклонения от индекса**:

```
отклонение_ноги = (цена_ноги / индекс − 1) × 100   [%]
индекс = нулевая ось.  Ноги дышат над/под нулём.
```

---

## Два паттерна осей

```
1. VALUE-CENTERED (проще, начни с него):
   ось X = время, ось Y = % отклонения от индекса (0 = индекс)
   ноги = линии, дышащие вокруг нуля. Видно ГДЕ и КОГДА расходится.
   → базис-веер, форвард-точка во времени

2. CURVE (снимок по сроку):
   ось X = дни до экспирации, ось Y = % от индекса
   точки = форварды серий. Видно ФОРМУ терм-структуры (контанго/бэквордация).
   → форвард-кривая

3. RADIAL / ORBIT (экзотика, если захочешь увидеть всю «семью» разом):
   центр = индекс, радиус = |отклонение|, угол = экспирация
   планеты-ноги на орбитах. Красиво, но читать труднее линий. Для дашборда.
```

Совет: начни с (1) — это то, что нужно в 90% случаев. (2) — для терм-структуры синта.
(3) — когда захочешь один общий «снимок семьи» на стену.

## Математика

```
отклонение ноги:   dev = (leg/index − 1) × 100
синт-форвард:       synth = K + Cmid − Pmid   (ATM-страйк у индекса)
синт-отклонение:    (synth/index − 1) × 100
implied carry год:  dev × 365 / DTE_дней
фандинг-эквив:      carry / (DTE × 3)   ← сравнить с реальным фандингом перпа
Z (выброс/норма):   (dev − μ) / σ   по окну
```

## Идеи графиков (с фокусом на синт и схождение)

```
A. БАЗИС-ВЕЕР (value-centered)        [построен]
   ось X=время · все ноги (перп/фьюч/синт) как dev от индекса
   → видно, какая нога дышит (фьюч ±0.3%, перп прижат ±0.02%) = где сетка

B. ФОРВАРД-КРИВАЯ (curve)             [построен]
   ось X=DTE · синт-форвард каждой серии как % от индекса
   → форма терм-структуры: бэквордация вблизи → контанго вдаль (carry ~+2%/год)
   → излом серии = аномалия для входа

C. ВЫПУКЛОСТЬ-КАРТА (payoff)
   ось X=цена индекса · payoff инверс(1/S) vs линейн(прямая)
   → показывает, есть ли реальная кривизна (мы выяснили: в USD её НЕТ)

D. ГРЕКО-ПРОФИЛЬ конструкции (payoff)
   ось X=цена индекса · Δ и Γ всей книги как функция цены
   → где конструкция нейтральна, где перекос (= payoff.py)

E. СКОС IV (curve вокруг ATM)
   ось X=страйк (центр=индекс) · IV каждого страйка
   → улыбка/перекос волы; для выбора страйка синтетики и краёв

F. ФАНДИНГ vs CARRY (value-centered)
   ось X=время · перп-фандинг-implied-базис vs синт-implied-carry
   → когда дешевле держать лонг перпом, когда синтетикой
```

## Рецепт-шаблон (скопировать, поменять символы)

```python
# 1. тянешь индекс + ноги (kline)
idx = kline('index-price-kline','ETHUSDT')      # {ts: price}
leg = kline('kline','ETHUSDT-19JUN26')          # фьюч/перп
# 2. отклонение
dev = {t: (leg[t]/idx[t]-1)*100 for t in idx if t in leg}
# 3. рисуешь dev во времени, ноль = индекс
```

```

```
